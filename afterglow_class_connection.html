Why CLASS only needs a derivative

CLASS is an ODE integrator. It holds a state vector pvecback_B[], and at each step it asks every fluid module:

"Given the current state y, what is dy/d(log a)?"

For the afterglow module, two slots are ours:

y[index_bi_rho_X] ← ρ_X (dark energy density) y[index_bi_Sigma] ← Σ (memory variable)

The glue API is exactly three functions:

afterglow_glue_initial_conditions()  // called ONCE, sets y at a_ini
afterglow_glue_derivs()              // called EVERY STEP, returns dy/dlog a
afterglow_glue_pressure()            // called when Friedmann needs p_tot

Everything else — $\Psi(r)$, $r = \rho_c/\rho_X$, $\beta$, $c_D$, the exchange current $Q$ — lives inside that one derivative call as intermediate arithmetic. CLASS never sees them.

The one-sentence takeaway: dρ_X/dloga is THE connection because CLASS is an integrator and integrators consume derivatives. Eq. 43 is the ODE. The physics lives in that one scalar return value.

Hierarchy of importance at the glue interface

1
dρ_X/dloga — called every step, encodes Eq. 43. If wrong, everything is wrong. Test 7 in test_afterglow_bg.py pins this point-wise.
2
dΣ/dloga — same ODE structure (Eq. 38). But Σ doesn't enter Friedmann, so bugs here are silent in H(a).
3
ρ_X (state, not derivative) — written once at IC, read every step for ρ_tot. The IC bug (missing pow(pba->H0,2)) was a units mismatch on this read.
4
p_X (via w_X) — algebraic: $p_X = w_X \rho_X$, $w_X = -1 + 1/(3c_D)$. Trivial.
5
Ψ(r), r, β, c_D, Q — internal to the derivative call. CLASS never sees them.
