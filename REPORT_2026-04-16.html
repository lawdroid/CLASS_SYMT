Step 4: Eq. 44 Back-Reaction Wiring

This was the main engineering work of the session. The PHASE1B2_PATCHES.md described a "two-line fix," but the reality required five edits because stock CLASS computes CDM analytically.

The problem

Stock CLASS: rho_cdm = Omega_cdm * H0^2 * a^(-3) <-- analytic, no ODE Eq. 44 needs: d(rho_cdm)/dloga = -3*rho_cdm - (beta/c_D) * Psi(r) * rho_X ^base ^back-reaction source There was no index_bi_rho_cdm in the integration vector. CDM had to be promoted from an algebraic quantity to an ODE variable.

Bug caught during testing

The beta != 0 gate: Without it, the ODE integrator introduces ~2e-4 numerical drift vs the exact analytic formula, even when the back-reaction term is zero. The gate ensures that beta=0 uses the analytic formula (bit-identical to pre-patch), and only beta != 0 activates the ODE path.

Eq. 55   Free-run limit: rho_X(a) = Omega_X * a^(-1/c_D) ▶

When beta_aft = 0 (no DM-DE coupling), the afterglow fluid decays as a pure power law in the scale factor. This is the free-run limit — the fluid relaxes under its own internal viscosity without any external exchange.

Eq. 43   Afterglow evolution: d(rho_X)/dN = -(1/c_D)[1 - beta*Psi] rho_X ▶

This is the master equation for the afterglow energy density. N = ln(a) is the number of e-folds.

When beta = 0, this reduces to d(rho_X)/dN = -(1/c_D) rho_X, whose solution is Eq. 55.

When beta != 0, the Psi(r) kernel modulates the decay rate depending on the ratio r = rho_cdm/rho_X.

A1: The PHASE1B2_PATCHES.md assumed index_bi_rho_cdm already existed in CLASS's integration vector. It doesn't. Stock CLASS computes CDM analytically as rho_cdm = Omega_cdm * a^(-3) — it's never integrated as a differential equation. To add a source term to something, that something must first be a variable in the ODE system. Promoting it required: (1) declaring the index, (2) allocating it, (3) setting initial conditions, (4) writing the base ODE, and (5) making background_functions() read from the ODE instead of the analytic formula.
A2: When beta = 0, the CDM source term is exactly zero (Test 10 verifies this). So the CDM ODE reduces to d(rho)/dN = -3*rho, whose solution is rho = rho_0 * a^(-3) — identical to the analytic formula. BUT numerical integration accumulates ~2e-4 relative error over cosmic time. By gating on beta != 0, we use the exact analytic formula when the back-reaction is absent, preserving bit-identity with pre-patch runs. This is important for regression testing and for ensuring that beta=0 is truly equivalent to the non-interacting limit.
A3: CLASS is compiled with -pthread (threading). When multiple threads compute different parts of the integration simultaneously, floating-point additions can happen in different orders across runs. Since floating-point addition is not associative ((a+b)+c != a+(b+c) at machine precision), identical inputs can produce outputs that differ at the ~1e-4 level. This is normal and expected in threaded numerical codes. It means "bit-identical" comparisons only work for the non-threaded code paths (like LCDM with analytic CDM).
A4: beta=0.1 with c_D=1.0 produces an extreme shift — this is well outside observational bounds and was used as a smoke test to confirm the back-reaction is physically active, not as a realistic parameter choice. Observationally viable values would be much smaller (beta ~ 0.01, c_D ~ 1.3), producing few-percent-level shifts. The Phase 3 MCMC will map the allowed region in {c_D, beta_aft} parameter space using Planck + DESI DR2 + Pantheon+ data.
A5: Phase 1 (background) determines how the average energy densities evolve — the homogeneous universe. Phase 2 (perturbations) determines how small fluctuations grow — the seeds of structure formation. Without perturbations, we can't compute the CMB power spectrum or matter power spectrum from first principles. Currently CLASS uses its stock perturbation equations for all species; Phase 2 adds three new degrees of freedom (delta_X, theta_X, sigma_hat) with the Maxwell-Israel-Stewart memory-dependent closure, which propagates the afterglow physics into the angular power spectrum. This is required before MCMC can constrain the model against real data.
A6: Yes — it's excellent. Machine epsilon for double precision is ~1.1e-16. A residual of 8.47e-17 means the Bianchi identity (energy conservation in the dark sector) is satisfied to below machine precision. This is because the cancellation is algebraically exact: the source term added to CDM is the negative of the exchange term in the afterglow equation, and both use the same Psi(r) computation. The residual comes only from floating-point rounding in the subtraction.

delta_X = delta(rho_X) / rho_X — the fractional density perturbation.

Just like CDM has delta_cdm (overdense/underdense regions), the afterglow fluid has delta_X. Where delta_X > 0, there's slightly more dark energy than average. Where delta_X < 0, slightly less.

Key insight: The sigma_hat - delta_X term is the MIS memory departure. When sigma_hat = delta_X (adiabatic), the fluid behaves like a standard barotropic fluid with sound speed cs2.
theta_X — the velocity divergence (how fast the fluid flows).

In cosmological perturbation theory, theta = div(v) captures the flow pattern. Positive theta = outflow (expansion), negative = inflow (collapse).

delta_p_X = w_X * rho_X * sigma_hat   (NOT tied to delta_X!)

The NaN/Inf Detective Story

Exit code 139 — Segmentation fault

After wiring all three perturbation equations into perturbations.c, we ran:

$ ./class test_phase2_beta01.ini Computing background... OK Computing thermodynamics... OK Computing perturbations... Segmentation fault (core dumped) Exit code: 139 (SIGSEGV)
Stack trace pointed to:
evolver_ndf15.c:1355 — numjac()
nj_ws->ydel_Fdel[nj_ws->Rowmax[j]][j]

This is CLASS's NDF15 implicit integrator computing a numerical Jacobian. The array index Rowmax[j] had been corrupted by NaN values in the perturbation vector.

Key observation: The crash was NOT in our afterglow code. It was deep inside CLASS's ODE solver. This means the output of our equations was the problem, not a coding bug like a null pointer.

Bisecting: which output mode triggers the crash?

CLASS computes different things depending on what you request. We systematically narrowed down which combination crashes:

Testoutput =lensingResult

Deduction: Lensing requires CLASS to integrate perturbations at many more k-modes (small-scale wavenumbers). The crash only happens at high k — this points to a scale-dependent instability. Small-scale (large k) perturbations blow up while large-scale ones stay fine.
Why does lensing need more k-modes?

Gravitational lensing of the CMB is caused by matter between us and the last scattering surface. Computing the lensing potential requires integrating structure on small scales (high k), which CLASS skips when you only ask for unlensed temperature (tCl).

Root cause: Jeans instability from negative sound speed

The velocity equation (P2) had the equation-of-state parameter w_X where the sound speed should go:

ORIGINAL equation (P2) — UNSTABLE: theta_X' = -H*(1 - 3*w_X)*theta_X + k^2*(w_X/(1+w_X))*sigma_hat

For the afterglow fluid with c_D = 1:

The math that kills it
w_X = -1 + 1/(3*1) = -2/3
1 + w_X = 1/3
w_X/(1+w_X) = (-2/3)/(1/3) = -2
What -2 means physically
Pressure gradient term: k^2 * (-2) * sigma
Wrong sign! Instead of resisting compression, pressure amplifies it.
Growth rate ~ e^(k*t) — faster for higher k.
The physics analogy

Imagine squeezing a balloon (positive sound speed: it pushes back). Now imagine a material where squeezing makes it collapse harder — that's negative effective sound speed. Sound waves become exponentially growing modes. This is the Jeans instability, normally associated with gravity overwhelming pressure, but here it's the equation itself producing a negative restoring force.

Technically: w < 0 means negative pressure (dark energy!). But the propagation speed of perturbations is a separate quantity. Sound speed squared must be positive for stable wave propagation. Using w as the sound speed gives cs^2 = w < 0 ⇒ imaginary sound speed ⇒ exponential blowup.

Why didn't this crash at low k? The instability growth rate scales as k^2. At k ~ 0.05/Mpc (CMB scales), the e-folding time is longer than the age of the universe. At k ~ 2/Mpc (lensing scales), perturbations double every ~10^(-2) conformal-time units — hitting 10^303 in milliseconds of integration.

The fix: separate sound speed from equation of state

The key insight: for non-adiabatic fluids, the equation of state w and the rest-frame sound speed cs2 are independent. CLASS's own fld module does exactly this with cs2_fld (defaults to 1, not w).

FIXED equation (P2) — STABLE: theta_X' = -H*(1 - 3*cs2_X)*theta_X + k^2*(cs2_X/(1+w_X))*sigma_hat With cs2_X = 1 (speed of light — the natural propagation speed in the hidden YM sector): cs2_X/(1+w_X) = 1/(1/3) = +3 ← positive! Waves propagate, don't explode.

Before the NaN crash, there was a linker error: CLASS compiles perturbations.c as C++ (.opp object) but afterglow files as C (.o objects). The C++ compiler mangles function names (e.g., afterglow_pert_rhs becomes _Z18afterglow_pert_rhsP...), so the linker couldn't match them.

A7: The fld module uses a barotropic closure: delta_p = cs2 * delta_rho. This means the pressure perturbation is algebraically determined by the density perturbation. The afterglow model's MIS closure is fundamentally different: delta_p = w * rho * sigma_hat, where sigma_hat is an independent dynamical variable with its own evolution equation. The stock fld module has no way to track this extra degree of freedom. Forcing afterglow into fld would destroy the memory physics that IS the whole point of the model.
A8: Think of the YM condensate as a viscoelastic substance. When you suddenly stretch a rubber band, the stress doesn't instantly equilibrate — it takes time. The memory sigma_hat tracks how far the pressure is from its equilibrium value. In Navier-Stokes, bulk stress is algebraically proportional to the expansion rate. In MIS theory, bulk stress has its own relaxation time (c_D / H). The background Sigma(a) captures the average memory; sigma_hat captures the perturbation memory. When sigma_hat = delta_X, the fluid is "relaxed" (adiabatic). Departures from this equality are the diagnostic fingerprint of MIS physics.
A9: The Euler equation has a pressure gradient term k^2 * (sound speed) * sigma_hat. When "sound speed" = w_X < 0, this becomes negative — pressure doesn't resist compression, it amplifies it. This is the cosmological Jeans instability with imaginary sound speed. Perturbations grow exponentially as ~exp(k * sqrt(|w|) * tau). At high k (small scales), the growth rate is enormous. The perturbation values reach 10^303 in a few timesteps, go to NaN, and the NDF15 integrator crashes when its numerical Jacobian encounters a NaN row.
A10: No — it's the correct physics. The equation of state w = p/rho determines the background pressure-to-density ratio. The sound speed cs2 = dp/drho determines how perturbations propagate. These are the same quantity only for a perfect adiabatic fluid. For a non-adiabatic fluid (which the afterglow IS, by construction), they differ. In the hidden YM sector, perturbations propagate at the speed of light (cs2 = 1) even though the background EoS is w = -2/3. This is analogous to water: the EoS gives p ~ rho, but sound waves travel at ~1500 m/s, much faster than the "EoS speed". CLASS's own fld module makes the same distinction with the cs2_fld parameter.
A11: Low-l multipoles (l < ~30) correspond to super-horizon scales that are dominated by the Integrated Sachs-Wolfe (ISW) effect. ISW is driven by the time-derivative of the gravitational potential, which directly depends on how dark energy dilutes. With w = -2/3 (c_D=1), dark energy dilutes much faster than Lambda, so the gravitational potential changes more rapidly — a huge ISW signal at low l. High-l multipoles (l > ~200) correspond to the acoustic peaks, set by physics at recombination (z ~ 1090). Dark energy was negligible then (Omega_DE ~ 10^{-9}), so the acoustic peaks barely change. This low-l / high-l pattern is the generic fingerprint of modified dark energy.
A12: sigma_hat starts equal to delta_X (adiabatic IC, the barotropic sub-manifold). Over time:

When beta=0: sigma_hat tracks delta_X closely; departures are driven only by the -(theta+h'/2)/(3c_D) term in (P3). The MIS memory signature is subtle.

When beta != 0: The coupling term (H/c_D)*beta*r*Psi'*(delta_c - delta_X) in (P3) drives sigma_hat away from delta_X whenever CDM and afterglow perturbations differ. This creates a larger MIS memory departure — the non-adiabatic pressure perturbation grows, leaving a distinctive imprint in the CMB polarization and in the matter power spectrum shape.
A13: Phase 3 MCMC runs a Markov Chain Monte Carlo sampler (Cobaya) that calls CLASS thousands of times with different parameter values {c_D, beta_aft, H0, omega_b, ...}, comparing each model's Cl and P(k) to real data (Planck CMB, DESI BAO, Pantheon+ supernovae). What Phase 2 provides is the complete forward model: given any parameter point, CLASS now computes both the expansion history (Phase 1) AND the perturbation growth (Phase 2), producing the full theoretical Cl(l) and P(k). Without Phase 2, the MCMC would only constrain the background (H(z), distance measures) and miss all information from the perturbation shape (acoustic peaks, ISW, matter clustering).
